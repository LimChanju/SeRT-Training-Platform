# 회고문

## 프로젝트 목표

이 프로젝트의 목표는 인간과 로봇이 함께 작업하는 협업 환경에서
데이터를 수집할 수 있는 플랫폼을 구축하는 것이었다.

일반적인 소비자용 웹 애플리케이션을 만드는 대신, VR 입력, Isaac Sim
로봇 시뮬레이션, 안전 관련 이벤트 로깅, trajectory 수집, AI 정책 평가를
연결하는 연구용 오픈소스 플랫폼을 만드는 데 초점을 두었다.

최종 프로젝트 범위는 다음과 같다.

- 인간-로봇 협업 시뮬레이션 환경에서 상호작용 데이터 수집
- 사람 손, 로봇, 큐브, 근접, 충돌 이벤트 로깅
- 안전과 관련된 상호작용 이벤트를 피드백 라벨로 변환
- BC/PPO 정책 학습에 사용할 수 있는 trajectory 데이터 구조 정리
- 학습된 로봇 정책의 rollout 평가 결과를 JSON/CSV로 저장
- README, CI/CD, 테스트, 보안 스캔, 헬스체크 API, 관측성 대시보드,
  릴리스 태그 등 OSS 제출 요건 정리

## 잘된 점

프로젝트가 진행되면서 플랫폼의 구조가 점점 명확해졌다. 초기에는
Isaac Sim 환경을 실행하고 로봇을 제어하는 데 초점이 있었지만, 이후에는
데이터 수집, 로그 저장, trajectory schema, 정책 학습, rollout 평가까지
연결되는 연구 파이프라인으로 확장되었다.

가장 의미 있었던 점은 시뮬레이션 실행 결과가 재현 가능한 데이터
산출물로 남도록 만든 것이다. 실행 중 발생하는 이벤트는 CSV 로그로
저장되고, rollout 평가 결과는 JSON/CSV 파일로 남으며, trajectory schema
문서를 통해 이후 imitation learning이나 reinforcement learning에 어떻게
활용할 수 있는지 정리하였다.

또한 저장소를 OSS 제출물에 가깝게 정리한 점도 의미가 있었다.
기본 커뮤니티 문서, CI 워크플로, Dependabot 설정, 보안 스캔 워크플로,
테스트, 헬스체크 API, 메트릭 대시보드 산출물을 갖추면서, 단순히 로컬에서
실행되는 연구 코드가 아니라 GitHub 링크로 검토 가능한 프로젝트 형태가
되었다.

## 어려웠던 점

가장 어려웠던 점은 이 프로젝트가 일반적인 웹 서비스가 아니라 연구용
데이터 수집 플랫폼이라는 점이었다. 최종 과제에서는 AI OSS 프로젝트로서
배포, 헬스체크, CI/CD, 테스트, 보안, 관측성, 문서, 릴리스, 데모 영상을
요구했기 때문에, 이 요구사항들을 Isaac Sim 기반 데이터 수집 플랫폼에
맞게 해석하고 연결하는 과정이 필요했다.

또 다른 어려움은 구현된 기능과 향후 연구 목표를 명확히 구분하는
것이었다. 현재 저장소에는 사람-로봇 근접 및 충돌 이벤트를 기반으로 안전
관련 피드백 라벨을 기록하는 경로가 구현되어 있다. 그중 내가 특정하게
다룬 형태가 pseudo-ErrP이지만, 이를 실제 EEG 기반 ErrP 시스템이 완성된
것처럼 표현해서는 안 된다. 현재 구현의 정확한 의미는, 향후 실제 EEG,
온라인 인간 피드백, 또는 다른 HRI safety signal로 대체 가능한 데이터 수집
및 피드백 라벨링 경로를 먼저 구현하고 검증했다는 것이다.

시뮬레이션과 정책 평가 스택을 맞추는 것도 쉽지 않았다. 로봇 제어,
그리퍼 동작, rollout 평가, reward shaping, 로깅이 서로 맞아야 수집된
데이터와 정책 평가 결과를 신뢰할 수 있기 때문에 여러 세부 조정이
필요했다.

## 배운 점

AI OSS 프로젝트는 모델이나 알고리즘만으로 완성되는 것이 아니라는 점을
배웠다. 특히 연구 플랫폼에서는 주변 엔지니어링 요소가 매우 중요했다.

- 명확한 데이터 흐름 문서
- 재현 가능한 평가 산출물
- CI/CD와 테스트 게이트
- 헬스체크 및 메트릭 엔드포인트
- 보안 자동화
- 릴리스 관리
- 평가자가 빠르게 이해할 수 있는 README

또한 연구 내용을 설명할 때는 주장 범위를 정확히 쓰는 것이 중요하다는
것을 배웠다. 현재 구현을 실제 EEG 기반 피드백이라고 과장하기보다,
안전 관련 상호작용 데이터를 수집하고 그중 일부를 pseudo-ErrP 형태의
피드백 라벨로 기록했다고 명확히 설명하는 것이 프로젝트의 신뢰도를 높이고
이후 확장 가능성도 더 잘 보여준다.

## 향후 개선 방향

향후에는 현재 pseudo-feedback 인터페이스를 실제 인간 피드백 소스와
연결하는 방향으로 발전시킬 수 있다.

개선하고 싶은 항목은 다음과 같다.

- 실제 EEG/ErrP 또는 온라인 인간 피드백 연동
- 더 현실적인 HRI stress test를 위한 human replay 데이터 확장
- 사람 개입 상황에서의 정책 학습 안정성 개선
- trajectory 품질과 안전 이벤트를 볼 수 있는 대시보드 확장
- rollout 평가에 대한 통합 테스트 보강
- Isaac Sim을 직접 실행하기 어려운 평가자를 위한 안정적인 데모 산출물
  제공

## 최종 소감

이 프로젝트는 랩실 연구를 위한 데이터 수집 환경에서 시작했지만, 최종적으로
AI OSS 제출 기준에 맞게 정리되었다. 핵심 기여는 인간-로봇 협업
시뮬레이션, 안전 관련 데이터 로깅, AI 정책 평가를 하나의 저장소 안에서
연결한 플랫폼을 만든 것이다.

가장 중요한 결과는 로봇 정책을 학습하거나 평가할 수 있다는 점만이 아니라,
그 데이터 수집 과정 자체가 문서화되고, 테스트 가능해지고, 관측 가능해지고,
GitHub를 통해 공유 가능한 형태로 정리되었다는 점이다.
