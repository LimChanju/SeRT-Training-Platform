# RL Trajectory Schema

This document defines the first learning data contract for robot policy training.
The v0 goal is state-based pick-and-place learning with binary pseudo-ErrP
feedback, while keeping the interface compatible with later EEG replay.

## Observation V0

Observation version: `obs_v0_state`

The canonical implementation lives in `v2/rl/observations.py`. The recorder
should save the dictionary fields below and may additionally save the flattened
policy vector generated by `flatten_observation()`.

| Field | Shape | Meaning |
|---|---:|---|
| `robot_joint_pos` | `[7]` | Panda arm joint positions, excluding the two gripper finger joints. |
| `robot_joint_vel` | `[7]` | Panda arm joint velocities. |
| `gripper_width` | `[1]` | Sum of the two Franka finger joint positions. |
| `ee_pos` | `[3]` | End-effector world position. |
| `ee_quat` | `[4]` | End-effector world orientation quaternion. |
| `cube_pos` | `[3]` | Current red pick cube world position. |
| `cube_quat` | `[4]` | Current red pick cube world orientation quaternion. |
| `cube_lin_vel` | `[3]` | Current red pick cube linear velocity. |
| `cube_ang_vel` | `[3]` | Current red pick cube angular velocity. |
| `place_target_pos` | `[3]` | Desired placement target world position. |
| `ee_to_cube` | `[3]` | Relative vector `cube_pos - ee_pos`. |
| `cube_to_place_target` | `[3]` | Relative vector `place_target_pos - cube_pos`. |
| `ee_to_place_target` | `[3]` | Relative vector `place_target_pos - ee_pos`. |
| `human_head_pos` | `[3]` | Human head or HMD world position. |
| `human_left_hand_pos` | `[3]` | Human left hand world position. |
| `human_right_hand_pos` | `[3]` | Human right hand world position. |
| `ee_to_left_hand` | `[3]` | Relative vector `human_left_hand_pos - ee_pos`. |
| `ee_to_right_hand` | `[3]` | Relative vector `human_right_hand_pos - ee_pos`. |
| `min_hand_gripper_dist` | `[1]` | Minimum distance from gripper center to either hand. Missing hands use `10.0m`. |
| `human_robot_collision` | `[1]` | Binary flag for human/robot collision. |
| `near_human` | `[1]` | Binary flag for unsafe proximity to a human hand. Default threshold is `0.12m`. |
| `collision_green` | `[1]` | Binary flag for collision with protected green cubes. |
| `pick_miss_recent` | `[1]` | Binary flag for a recent failed grasp attempt. |
| `drop_throw_recent` | `[1]` | Binary flag for a recent drop or throw event. |
| `has_grasped_cube` | `[1]` | Binary flag for whether the active cube is currently grasped. |
| `task_phase` | `[4]` | One-hot phase: `approach_cube`, `grasp_cube`, `move_to_target`, `release_cube`. |

Flattened dimension: `73`.

## Action V0

The first recorder should store both action forms when possible:

| Field | Shape | Meaning |
|---|---:|---|
| `expert_task_action` | `[5]` | End-effector command: `dx, dy, dz, dyaw, gripper_cmd`. |
| `expert_joint_action` | `[9]` | Raw articulation action from the expert controller. |

Policy training can start from `expert_task_action`, while `expert_joint_action`
remains useful for replay and debugging.

## Reward V0

The reward should be stored per step:

```text
 distance improvement: ee -> cube
+ distance improvement: cube -> target
+ grasp bonus
+ success bonus
- action penalty
- near human penalty
- human collision penalty
- ErrP feedback penalty
```

At first:

```text
errp_feedback = errp_label
```

With EEG replay:

```text
errp_feedback = eeg_classifier(sample(error_or_non_error_epoch))
```

## ErrP V0

Store ErrP separately from reward so the same trajectories can be replayed with
different feedback models.

| Field | Shape | Meaning |
|---|---:|---|
| `label` | `[T, 1]` | Binary pseudo-ErrP label. |
| `feedback` | `[T, 1]` | Feedback value used by reward. Initially `0.0/1.0`, later EEG classifier probability. |
| `source_code` | `[T, 1]` | Numeric code for the event source. |
| `event_step` | `[T, 1]` | `1` only on the timestep where the event starts. |
| `eeg_replay_used` | `[T, 1]` | `1` if feedback came from EEG replay. |
| `eeg_epoch_id` | `[T]` | Optional identifier for replayed EEG epoch. |
