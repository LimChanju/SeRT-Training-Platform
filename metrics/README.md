# DORA Metrics (Experiment Pipeline)

This repository tracks DORA-style metrics tailored for the VR human-robot collaboration data pipeline.

## Metric Definitions
- **Lead Time**: Time from scenario/code change to successful data episode extraction.
- **Deployment Frequency**: Number of successful data sessions per day.
- **Change Failure Rate**: Failed sessions / total sessions.
- **MTTR**: Time to recover from pipeline failures (incident start to end).

## Event Log Schema
`metrics/session_events.csv`

Columns: `timestamp,event,details`

Example events:
- `code_change`
- `session_start`
- `session_success`
- `session_failed`
- `incident_start`
- `incident_end`
